<static> {boolean} JKIL.types.isString(pm_value, ph_options)
Checks whether the given value is a string litteral, a String object, or either a string litteral or a String object.
       Syntax:
                JKIL.types.isString ( pm_value[, ph_options] )
       Example 1 - Synonym of JKIL.types.isStringLitteral(): 
                var v = "a string";
                var b = JKIL.types.isStringLitteral ( v, {litteral: true, object: false} ); // true
       Example 2 - Synonym of JKIL.types.isStringObject(): 
                var v = new String("a string");
                var b = JKIL.types.isStringLitteral ( v, {litteral: false, object: true} ); // true
       Example 3 - Synonym of JKIL.types.isStringObjectOrLitteral(): 
                var v1 = "a string";
                var b = JKIL.types.isStringLitteral ( v1, {litteral: true, object: true} ); // true
                var v2 = new String("a string");
                var b = JKIL.types.isStringLitteral ( v2, {litteral: true, object: true} ); // true
Parameters:
{mixed} pm_value
The value whose type is to be checked.

{hash} ph_options Optional
A hash of options:

{boolean} ph_options.litteral Optional, Default: true
If true, the method returns true for string litterals. ph_options.litteral and ph_options.object cannot be both false.

{boolean} ph_options.object Optional, Default: true
If true, the method returns true for String objects. ph_options.litteral and ph_options.object cannot be both false.
Returns:
{boolean} Returns true if the given value is strictly a string litteral, or strictly a String object, or either a string litteral or a String object. Returns false otherwise.
Requires:
JKIL.types.getClass()
